[0, 100]
We combined two different SHA256 hashes and transformed the resulting hash into a number that lies within a uniform, random distribution from [0,100]. A series of games to be played is generated in a large chain, where each element in the chain is the previous element passed through the hashing function.
10 million games were generated with this mechanism. The games are played in reverse order so the ten millionth result is Game 1 and so on. Anyone can verify that previous games match up with the corresponding hashed number with simple javascript in a browser.
